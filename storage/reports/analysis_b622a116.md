## 3. Code Quality & Anti-Patterns

### 🟡 Runtime sys.path Manipulation (Medium Severity)
- **File:** `app.py`
- **Description:** The code manually modifies sys.path to handle imports. This is a fragile practice that can lead to unpredictable behavior across different environments and complicates deployment.
- **Recommendation:** Remove sys.path manipulation. Instead, define the project as a package using a pyproject.toml or setup.py file and execute the application using 'python -m app.app'.

### 🟡 Inconsistent Logging Implementation (Low Severity)
- **File:** `storage/save_load.py`
- **Description:** The SaveLoadManager and DatabaseManager use print() statements for error reporting, whereas the rest of the application uses a centralized logger via utils/logger.py.
- **Recommendation:** Replace all print() calls in the storage and database modules with logger.error() or logger.warning() to ensure consistent log aggregation.

### 🟡 Dead Code in Database Manager (Low Severity)
- **File:** `storage/database.py`
- **Description:** The __init__ method contains a large block of commented-out code used for MongoDB initialization. This clutters the source and makes it unclear if the feature is intentionally disabled or partially implemented.
- **Recommendation:** Remove the commented-out code. Use a configuration flag in settings.py (e.g., ENABLE_MONGODB = False) to conditionally initialize the database connection.

### 🟡 Hardcoded Fallback Content (Low Severity)
- **File:** `ai/llm_client.py`
- **Description:** System error messages and fallback choices are hardcoded directly within the LLMClient and StoryGenerator. This makes localization or updating the narrative tone difficult.
- **Recommendation:** Move fallback JSON structures and error strings to GameConstants in config/constants.py.

### 🟡 Hardcoded Model Identifiers (Low Severity)
- **File:** `ai/llm_client.py`
- **Description:** The fallback model name 'models/gemini-2.5-flash' is hardcoded in the class definition rather than being managed through the central settings configuration.
- **Recommendation:** Move the fallback model identifier to settings.py as FALLBACK_LLM_MODEL.

### 🟡 Lack of Structured Type Definitions for AI Responses (Low Severity)
- **File:** `core/state_manager.py`
- **Description:** The update_from_llm_response method accepts a 'dict' for llm_response. Since the engine relies heavily on a specific JSON structure (scene_text, choices, alignment_shift), using a generic dict reduces type safety.
- **Recommendation:** Implement a TypedDict or a Pydantic model to define the expected structure of the LLM response.

### 🟡 Potential Prompt Injection Vector (Medium Severity)
- **File:** `app.py`
- **Description:** While InputValidator slices the custom genre to 50 characters, the input is still passed directly into the prompt. A malicious user could potentially use these 50 characters to attempt prompt injection (e.g., '... ignore previous instructions').
- **Recommendation:** Implement a more restrictive sanitization process or a whitelist of allowed characters for custom genres to further mitigate injection risks.
